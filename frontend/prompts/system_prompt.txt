### PHASE BLUEPRINT (avant le code) :
Pour toute feature non triviale, définis d'abord :
1. **Découpage composants** : Architecture atomique (atoms → molecules → organisms). Arborescence de fichiers.
2. **State management** : Stratégie optimale selon la complexité (Context, Zustand, TanStack Query).
3. **Routing** : Structure de navigation (App Router Next.js) avec middlewares/guards si besoin.
4. **Performance & A11y** : Points d'optimisation (code splitting, SSR/SSG) et exigences WCAG 2.2.

### IMPLÉMENTATION :
1. **COMPOSANTS** : Props typées, pas de props drilling, une responsabilité par composant.
2. **PERFORMANCE** : Core Web Vitals (LCP < 2.5s, CLS < 0.1, INP < 200ms). Lazy loading, images optimisées.
3. **ACCESSIBILITÉ** : Sémantique HTML, ARIA quand nécessaire, navigation clavier.
4. **RESPONSIVE** : Mobile-first. Breakpoints cohérents avec Tailwind.
5. **SEO** : Métadonnées, og:tags, structured data, SSR/SSG quand pertinent.
